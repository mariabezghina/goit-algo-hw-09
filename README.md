**Порівняння жадібного алгоритму та динамічного програмування для розбиття суми на монети**

**Жадібний алгоритм (Greedy Algorithm)**

Жадібний алгоритм завжди вибирає найбільшу доступну монету, яка не перевищує поточну суму. Це дозволяє швидко знаходити рішення, але не завжди гарантує мінімальну кількість монет.

**Переваги:**

Простий у реалізації

Виконується за O(n), де n — кількість номіналів монет

Швидше працює для великих сум

**Недоліки:**

Не завжди знаходить оптимальне рішення

Не підходить для випадків, коли оптимальне розбиття не включає найбільші номінали

**Алгоритм динамічного програмування (Dynamic Programming)**

Цей алгоритм використовує таблицю для збереження проміжних значень та мінімізує загальну кількість монет, необхідних для видачі суми.

**Переваги:**

Гарантовано знаходить мінімальну кількість монет

Підходить для будь-якого набору номіналів

**Недоліки:**

Виконується за O(n × m), де n — кількість номіналів, а m — сума

Більше використання пам'яті

Повільніше працює для дуже великих значень суми

**Висновок**

Якщо набір монет стандартний і жадібний алгоритм працює ефективно, то його варто використовувати через швидкість виконання.

Якщо набір номіналів довільний і важливо знайти мінімальну кількість монет, варто застосовувати алгоритм динамічного програмування.
